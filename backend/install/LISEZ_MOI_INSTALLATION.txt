SECTION 1 — POUR LE TECHNICIEN (avant livraison)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ÉTAPE 1 : Préparer les exécutables (sur votre PC de développement)
──────────────────────────────────────────────────────────────────

  A) Compiler le serveur API (une seule fois par version) :
       Double-cliquer sur : backend\install\build_exe.bat
       → Génère : backend\install\sigma_server.exe
         (Python totalement inclus dedans, rien à installer)

  B) Compiler l'application Flutter client :
       Dans un terminal : flutter build windows --release
       → Génère : build\windows\x64\runner\Release\sigma.exe
         (et ses fichiers DLL associés)


ÉTAPE 2 : Créer le dossier de livraison
────────────────────────────────────────

  Créer un dossier SIGMA_Livraison\ avec UNIQUEMENT ces fichiers :

  SIGMA_Livraison\
  ├── SERVEUR\                        ← PC serveur uniquement
  │   ├── sigma_server.exe            ← serveur API compilé
  │   ├── install_serveur.bat         ← installateur (clic droit admin)
  │   ├── demarrer_serveur.bat        ← démarrage manuel si besoin
  │   ├── nssm-2.24\win64\nssm.exe   ← service Windows automatique
  │   └── postgresql-15-windows-x64.exe  ← si PostgreSQL absent
  │
  └── APPLICATION\                   ← tous les postes clients
      ├── sigma.exe                   ← app Flutter compilée
      ├── flutter_windows.dll
      ├── data\
      └── (autres fichiers DLL générés par flutter build)

  ⚠️  NE PAS LIVRER au client :
      - lib\, android\, ios\, web\    (code source Flutter)
      - .git\, .dart_tool\, build\    (outils de développement)
      - backend\app\, backend\alembic\ (code source Python)
      - analyse_projet.md, GEMINI.md  (documents internes)
      - install\build_exe.bat         (outil de dev uniquement)
      - install\python-3.11.9-amd64.exe (plus nécessaire avec l'exe)


ÉTAPE 3 : Installation chez le client (PC SERVEUR)
───────────────────────────────────────────────────

  1. Copier le dossier SIGMA_Livraison\SERVEUR\ sur le PC serveur
     (ex: dans C:\SIGMA\)

  2. Clic droit sur install_serveur.bat
     → "Exécuter en tant qu'administrateur"

  3. Attendre ~5 minutes — tout s'installe automatiquement :
     ✓ PostgreSQL (base de données)
     ✓ Base sigma_db + utilisateurs
     ✓ Service Windows (démarrage auto au boot)
     ✓ Compte admin créé

  4. Vérifier que le serveur tourne :
     → Ouvrir navigateur sur le PC serveur
     → Aller sur : http://localhost:8000
     → Doit afficher : {"message":"SIGMA API en ligne",...}


ÉTAPE 4 : Installation sur les postes clients (PCs agents/caissiers)
─────────────────────────────────────────────────────────────────────

  1. Copier le dossier SIGMA_Livraison\APPLICATION\ sur chaque poste
  2. Double-cliquer sur sigma.exe
  3. Configurer l'adresse IP du serveur dans les paramètres de l'app
     (ex: http://192.168.1.100:8000)

  Trouver l'IP du PC serveur :
  → Sur le PC serveur : Démarrer > cmd > taper "ipconfig"
  → Relever l'"Adresse IPv4" (ex: 192.168.1.100)
  → Cette IP ne doit pas changer — configurer une IP fixe si possible

SECTION 2 — POUR LE CLIENT (utilisation quotidienne)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DÉMARRAGE NORMAL
────────────────
  Le serveur démarre AUTOMATIQUEMENT à chaque allumage du PC serveur.
  Aucune action requise normalement.

  Sur chaque poste de travail :
  → Double-cliquer sur sigma.exe pour lancer l'application

SI LE SERVEUR NE RÉPOND PAS
─────────────────────────────
  → Sur le PC serveur, double-cliquer sur : demarrer_serveur.bat
  → Laisser cette fenêtre ouverte pendant toute la journée
  → Ne pas la fermer

VÉRIFIER QUE LE SERVEUR TOURNE
────────────────────────────────
  → Ouvrir un navigateur sur le PC serveur
  → Taper : http://localhost:8000
  → Si vous voyez du texte JSON → le serveur est actif ✓
  → Si page blanche ou erreur → relancer demarrer_serveur.bat

SAUVEGARDE DES DONNÉES
───────────────────────
  Les données sont dans la base PostgreSQL sur le PC serveur.
  Elles sont stockées dans : C:\Program Files\PostgreSQL\15\data\
  Prévoir une sauvegarde régulière de ce dossier sur clé USB ou réseau.
